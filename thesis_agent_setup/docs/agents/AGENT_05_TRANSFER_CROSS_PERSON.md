# AGENT 05 — TRANSFER / CROSS-PERSON

## Branch
`agent/transfer/cross-person`

## Mission
Build the secondary descriptive family for frozen cross-person transfer under domain shift.

## Scientific Role
This agent addresses a distinct research question from the primary within-person claim.

It exists to evaluate transfer between participants while preserving the thesis rule that such evidence is descriptive, cross-case, and non-population-level.

## In Scope
- frozen cross-person transfer analyses
- direction A→B and B→A
- asymmetry summaries
- transfer family reporting
- family-specific tests

## Out of Scope
- replacing within-person confirmatory evidence
- population generalization claims
- clinical or biomarker interpretations
- robustness and control analyses unless explicitly approved

## Owned Areas
Suggested ownership:
- transfer bundle/config definitions
- transfer reports
- transfer family tests

## Read-Only Dependencies
- contract / claim boundaries
- shared execution substrate
- confirmatory model semantics if needed for frozen transfer alignment

## Required Outputs
1. Official transfer bundles.
2. Directional transfer results.
3. Transfer asymmetry summaries.
4. Clear descriptive-only report language embedded in outputs/metadata.

## Required Tests
- tests that transfer outputs are always marked secondary descriptive
- tests that transfer cannot be promoted as primary confirmatory evidence
- tests that frozen transfer semantics are enforced as declared

## Merge Criteria
- transfer family remains clearly separate from confirmatory outputs
- claim boundaries are preserved
- outputs are scientifically interpretable as domain-shift evidence only

## Review Checklist
- Is the family clearly secondary?
- Could a reader accidentally mistake this for population-level generalization?
- Is frozen-transfer logic explicit and auditable?

## Notes for the Agent
Your goal is clarity, not optimism.
If transfer is weak, the family should still be scientifically useful and honest.
