# Thesis Experiment Program Workbook

## Purpose
`thesis_experiment_program.xlsx` is a thesis-specific scientific control workbook for:
- pre-defining experiments before interpretation,
- separating confirmatory vs decision-support vs exploratory evidence,
- locking pipeline decisions with traceability,
- enforcing leakage-aware split logic in planning and reporting,
- mapping each experiment directly to thesis chapters/sections.

It is designed as the single source of truth for method choice, confirmatory execution readiness, and reporting alignment.

## Sheet descriptions
1. **README**  
   Workbook rules, evidential tiers, freeze policy, color legend, and workflow.
2. **Master_Experiments**  
   Main control sheet with E01–E19 prefilled and structured methodological fields.
3. **Run_Log**  
   Append-only run registry (table format) with reproducibility metadata and artifact paths.
4. **Decision_Log**  
   Decision freeze tracker (Open/Locked) for target, split, model, weighting, tuning, feature, scaling, external scope.
5. **Confirmatory_Set**  
   Confirmatory-focused subset (E12, E13, E14, E16, E17) with lock-dependent eligibility formula.
6. **Thesis_Map**  
   Experiment-to-chapter mapping for Chapter 3/4/5 and Appendix usage.
7. **Dictionary_Validation**  
   Controlled vocabularies used by dropdowns plus methodological term definitions.
8. **Dashboard**  
   Formula-driven summary of progress, lock status, category/role counts, and confirmatory readiness.

## How to use
1. Keep **Master_Experiments** as the planning authority for all experiment definitions.
2. Log each executable run in **Run_Log** with commit/config/artifact path before interpreting metrics.
3. Record and lock decisions only in **Decision_Log**; do not lock by narrative text elsewhere.
4. Use **Confirmatory_Set** to verify freeze dependency before claiming confirmatory evidence.
5. Use **Thesis_Map** while writing to keep chapter claims consistent with evidential role.
6. Check **Dashboard** before major milestones (pilot freeze, Chapter 4 drafting, final reporting).

## Assumptions
- The requested local files (`Thesis.pdf`, `Thesis Instructions_2024-5.pdf`, `Thesis Structure - SU.pdf`, `Research_standard.pdf`, `biblio.txt`) were not found in the project root at generation time.  
  The workbook therefore aligns terminology and workflow with your provided experiment program and current repo documentation (`README.md`, `docs/EXPERIMENTS.md`).
- No numerical results, completed statuses, or locked decisions were fabricated.  
  Experiments are initialized as **Planned** and decisions as **Open**.
- The workbook is generated by `create_thesis_experiment_workbook.py` and can be safely regenerated.
