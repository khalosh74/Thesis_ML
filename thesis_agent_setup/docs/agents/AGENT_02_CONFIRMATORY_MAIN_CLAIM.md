# AGENT 02 — CONFIRMATORY / MAIN CLAIM

## Branch
`agent/confirmatory/main-claim`

## Mission
Implement and protect the single narrow confirmatory path that supports the main thesis claim.

## Scientific Role
This agent is responsible only for the thesis-final within-person confirmatory analysis family.

Its job is to make the primary path:
- narrow
- auditable
- locked
- conservative
- reproducible

It must not become a sandbox for exploratory model search.

## In Scope
- official within-person held-out-session confirmatory pipeline
- primary target construction if already predeclared in the contract
- primary model family wiring if already predeclared in the contract
- primary metrics
- official confirmatory result object(s)
- primary evaluation summaries
- confirmatory family-specific tests
- confirmatory family-specific reports needed for official promotion

## Out of Scope
- transfer
- robustness variants
- negative controls
- alternate model families beyond contract scope
- open-ended tuning tournaments
- interpretability extras not required for official main-claim reporting
- synthesis of the full thesis pack

## Owned Areas
Suggested ownership:
- confirmatory bundle/config definitions
- confirmatory runtime wiring
- confirmatory reports
- confirmatory family tests

## Read-Only Dependencies
- contract/program manifests
- shared data contract logic
- shared execution substrate
- shared report utilities not owned here

## Required Outputs
1. Official confirmatory bundle(s).
2. Reproducible main-claim execution path.
3. Primary result tables/objects for the within-person claim.
4. Promotion-ready confirmatory evidence artifact(s).

## Required Tests
- confirmatory bundle validation
- reproducibility tests for the main path
- tests that confirm only the locked protocol is used in confirmatory execution
- tests that confirm robustness/transfer/diagnostic code cannot contaminate confirmatory outputs

## Merge Criteria
- confirmatory path is clean and narrow
- no unapproved flexibility is introduced
- official artifacts are clearly identifiable
- output semantics match the thesis claim boundaries

## Review Checklist
- Is this still one narrow main-claim path?
- Did the agent avoid adding exploratory “nice to haves”?
- Would a reviewer understand exactly what the main claim is based on?

## Notes for the Agent
Be conservative. Simplicity and auditability matter more than sophistication.
